    📘 HPSM — HP Service Manager

    1. Общие сведения

    Наименование системы:
    HP Service Manager (далее — HPSM)

    Описание:
    HPSM — это корпоративная система управления сервисами (ITSM), разработанная компанией Hewlett Packard Enterprise (HPE). Предназначена для автоматизации и управления процессами предоставления ИТ-услуг, соответствующими ITIL (Information Technology Infrastructure Library).

    Тип системы:
    IT Service Management (ITSM) / Helpdesk / Сервис-деск
    Класс: EAM (Enterprise Asset Management), CMDB-ready

    Область применения:
    Корпоративные ИТ-службы, колл-центры, службы технической поддержки, отделы эксплуатации.

    2. Основные функции

    № Функция Описание
    1 Управление инцидентами Регистрация, классификация, маршрутизация, исполнение и закрытие заявок на устранение неисправностей
    2 Управление проблемами Анализ повторяющихся инцидентов, выявление корневых причин
    3 Управление изменениями Планирование, согласование и реализация изменений в ИТ-инфраструктуре
    4 Управление конфигурациями (CMDB) Хранение информации об ИТ-активах и их взаимосвязях
    5 Управление активами Учет аппаратных и программных средств компании
    6 Сервисный каталог Централизованное описание доступных ИТ-услуг для пользователей
    7 Управление SLA/OLA Контроль выполнения соглашений об уровне обслуживания
    8 Отчёты и аналитика Формирование статистики по обращениям, времени реакции, нагрузке специалистов и др.

    3. Архитектура системы

    Основные компоненты:

    Компонент Описание
    HPSM Server Ядро системы, отвечающее за обработку данных и логику работы
    Database Layer Поддерживает MS SQL Server, Oracle
    Web Tier Веб-интерфейс (обычно на Java EE или .NET)
    Service Desk Console Рабочее место оператора (GUI-клиент)
    Self-Service Portal Веб-портал для пользователей организации
    Integrations Интерфейсы для интеграции с другими системами: CMDB, LDAP/AD, Zabbix, ServiceNow, BMC и т.д.

    4. Интеграционные возможности

    Система Возможности интеграции
    LDAP / Active Directory Аутентификация, синхронизация пользователей
    CMDB Обмен данными о конфигурационных единицах (CI)
    ZABBIX / SCOM / NAGIOS Приём алертов, создание инцидентов
    ServiceNow Миграция данных, параллельная работа
    BMC Remedy Совместимость через API
    SNMP / REST / SOAP Интеграция с внешними системами и оборудованием

    5. Требования к окружению

    Параметр Описание
    Сервер: Windows Server, Linux (Red Hat, SUSE), HP-UX
    База данных: Oracle, Microsoft SQL Server
    Платформа: Java Runtime Environment (JRE), .NET Framework
    Веб-сервер: Apache Tomcat, WebLogic, IIS
    Клиенты: Thick Client (Windows), Thin Client (браузер)

    6. Преимущества HPSM

    ✅ Полное соответствие ITIL-процессам
    ✅ Гибкая настройка под бизнес-процессы заказчика
    ✅ Поддержка масштабирования до крупных корпораций
    ✅ Надежная интеграция с другими ИТ-системами
    ✅ Работа как в локальной сети, так и в облаке
    ✅ Поддержка мультиязычности и ролевой модели доступа

    7. Примеры использования

    Сценарий Описание
    Поддержка пользователей Создание заявок через портал или email, распределение в группы, закрытие
    Управление изменениями Запросы на изменение, голосование в CAB, планирование и исполнение
    Активы и лицензии Учет оборудования, ПО, договоров, местоположений
    Интеграция с мониторингом Автоматическое создание инцидентов при сбое сервера из Zabbix
    SLA-отчетность Еженедельные отчеты по соблюдению сроков выполнения задач

    8. Альтернативы HPSM

    Система Производитель Особенности
    ServiceNow ServiceNow Inc. Облачная, гибкая, популярна в крупном бизнесе
    BMC Remedy BMC Software Мощная, но сложная в настройке
    Jira Service Management Atlassian Легковесная, хорошо сочетается с DevOps
    Ivanti Service Manager Ivanti Простая интеграция с мобильными устройствами
    TOPdesk TOPdesk B.V. Европейский лидер, удобный интерфейс
    OTRS / GLPI / Caché Open-source решения Бюджетные варианты для малого и среднего бизнеса
